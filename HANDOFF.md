# 작업 경과 요약 (CLI 세션 → 데스크탑 이관용)

`index.html` 웹앱을 어떻게 여기까지 만들었는지의 기록. 상세 스펙은 `CLAUDE.md` 참고.

## 어떻게 시작했나
1. clobe MCP 서버 추가(`claude mcp add --transport http clobe ...`) + OAuth 인증.
2. "계좌 잔액 / 어제 입출금 / 카드내역" 등을 MCP로 실제 조회하며 clobe 데이터 구조 파악.
   - 이 과정에서 회사=제이앤에이취프레스, 실제 계좌·거래·세금계산서·카드 데이터 확인.
3. 사용자가 "조회창을 웹앱으로 제작" 요청 → 목업 데이터 우선으로 단일 HTML 앱 제작 시작.

## 요청 → 반영 순서 (시간순)
1. **초기 앱**: 사이드바 5개 뷰(계좌잔액/입출금/카드청구/카드이용/세금계산서) + 개요, dataviz 팔레트, 다크모드.
2. **카드 사용자 매칭 + 기간 설정**: 카드번호↔사용자 이름 매칭(localStorage), 전역 기간 프리셋+직접지정.
3. **거래처 속성 + 엔티티 모달 + 계좌 별명**: 거래처에 거래유형/공정/비용유형 입력→필터, 거래처·계좌·카드 클릭 시 드릴다운 모달, 계좌 별명 매칭.
4. **헤더 고정 + 챗봇**: sticky 상단바, 우하단 자연어 조회 챗봇("어제 입출금 내역 알려줘" 등).
5. **챗봇 연도 인식 버그 수정**: "2026년" 같은 연/올해/작년 파싱 추가, 월말 날짜 정확화.
6. **명칭·통합·제약**: "거래처·매칭 관리"→"환경설정", 카드청구+이용→"카드 내역"(탭), 카드뷰에서 사용자매칭 제거, 거래처에 사업자번호·대표자·주소 추가, (일시적) 한 페이지 제약.
7. **전량표시 + 보유카드 전체 + 거래처 등록 테이블**: 페이지 제약 제거(기간 내 전부 표시), 사용이력 없는 카드도 표시, 거래처 속성→"거래처 등록" 테이블(헤더 정렬 + 검색).
8. **세금계산서 실데이터**: `get_tax_invoices`로 2026-06-25~07-01 실제 30건 시드.
9. **카드 실데이터**: `get_labeled_card_billing_items`로 실제 카드 11장·이용 25건·사용자명 자동 매칭 시드. 기본 조회기간 90일로.
10. **계좌·입출금 병합**: 계좌잔액+입출금 한 뷰로. 상단 8지표(예금/펀드/외화/대출/이월/입금/출금/기말), 거래유형·공정·비용유형을 필터 드롭다운에서 **테이블 컬럼**으로 이동, 계좌별 탭.

## 확인된 핵심 사실 (재확인 불필요)
- **세금계산서·현금영수증·매출·은행거래·카드청구는 clobe MCP로 실조회 가능.**
- **카드 "이용내역"은 별도 도구가 없지만**, `get_labeled_card_billing_items`가 건별로 카드번호·userNames·가맹점·사용일까지 주므로 사실상 대체 가능. 단 **결제일 기준**이라 하루 단위 조회는 0건 나오기 쉬움(청구일 통상 25일).
- 실제 보유카드는 BC(4140/9430/6250) 위주 + 신한(5589) 일부. 사용자명(조*성 등)은 신한카드에만 붙어 있었음.
- 데이터 최신화(재수집)는 MCP 불가 → app.clobe.ai에서 사용자가 직접.
- 데이터 규모 큼(세금계산서 연 1,833건, 카드 4,850건) → 실연동 시 기간·페이지 분할 필수.

## 미해결 / 주의
- 이월·기말 잔액은 현재 목업 거래 몇 건으로 롤백 추정 → 실제와 다를 수 있음(실연동 시 정확).
- 은행 거래(transactions)·계좌·대출·월매출은 아직 상당수 목업. 세금계산서/카드만 실데이터 발췌.
- 남은 BC 카드들 사용자명 미매칭(환경설정에서 입력 가능).
